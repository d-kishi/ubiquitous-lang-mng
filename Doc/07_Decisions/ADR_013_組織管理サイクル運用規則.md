# ADR_013: 組織管理サイクル運用規則

**ステータス**: 承認済み  
**決定日**: 2025-07-16  
**決定者**: Claude Code  
**レビュー予定**: 2025-10-16  

## 📋 決定事項

### 組織管理サイクル運用規則の確立

Phase適応型組織化による構造化専門分析・実装において、組織運用の見落とし防止と品質保証を目的とした組織管理サイクル運用規則を確立する。

## 🎯 決定の背景

### 発生した問題

Phase A1 Step1の再実行において、組織管理サイクルの見落としが発生：
- Step終了時の組織レビューチェックリスト実施が漏れた
- 組織設計結果の適切な記録場所（Active配下）への記録が不十分
- 特殊条件下（Step再実行）での運用手順が不明確

### 根本原因

1. **文書構造の複雑化**: プロジェクト状況.mdが肥大化し、重要な運用ルールが埋没
2. **情報の分散配置**: 組織関連情報が複数文書に分散し、全体像把握が困難
3. **継ぎ足し構造**: 段階的追加による可読性低下
4. **注意喚起の不足**: 組織管理サイクルの重要性が十分に強調されていない

4. **ディレクトリ移動** → `Phase_XX/` 全体を Active → Completed へ移動
5. **次Phase準備** → 必要に応じて次Phase計画・技術基盤整理

### 組織設計原則

#### Phase特性による組織最適化
- **並列分析適用**: 課題発見・技術調査・アーキテクチャ検討段階
- **順次統合適用**: 実装・統合・テスト段階（依存関係あり）
- **専門特化適用**: 特定技術領域の集中解決段階

#### 組織継続 vs 変更の判断基準
- **作業性質**: 並列 vs 順次
- **専門性要求**: 深い専門知識 vs 統合調整
- **成果物特性**: 分析・設計 vs 実装・統合
- **課題状況**: 新規発見 vs 既知解決

## 🔧 技術実装

### ファイル管理体系

#### 組織設計ファイルの管理単位（2025-07-21更新）
**重要**: 組織設計ファイルは**Phase配下にStep毎ディレクトリ構造**で管理する

**新管理方針**:
- 1つのPhaseにつき1つのディレクトリ（例: Phase_A2/）
- Phase概要・総括を `Phase_Summary.md` で管理
- 各Step組織設計・レビューを `StepXX_[内容].md` で管理
- Phase完了時にPhase_XXディレクトリ全体をActive→Completedへ移動

#### 新ディレクトリ構造
```
/Doc/08_Organization/{Active|Completed}/Phase_XX/
├── Phase_Summary.md              # Phase全体概要・総括
├── Step01_Analysis.md            # Step1組織設計・レビュー記録（全Phase共通）
├── Step02_[Phase固有内容].md     # Step2組織設計・レビュー記録（Phase毎に決定）
├── Step03_[Phase固有内容].md     # Step3組織設計・レビュー記録（Phase毎に決定）
├── Step04_[Phase固有内容].md     # Step4組織設計・レビュー記録（Phase毎に決定）
└── StepXX_[Phase固有内容].md     # 各Step組織設計・レビュー記録（Phase毎に決定）
```

**重要**: Step02以降の内容・数・命名はPhase特性により決定される
- **Step01**: 全Phase共通で「Analysis」（並列分析・計画詳細化）
- **Step02以降**: Phase毎の作業特性に応じて内容・ファイル名を決定
- **例**: Phase A1なら「Step02_Implementation.md, Step03_Testing.md」
- **例**: Phase A2なら「Step02_Domain_Implementation.md, Step03_Infrastructure.md, Step04_Web_Implementation.md...」

#### Phase_Summary.md内容構造
```markdown
# Phase XX 組織設計・総括

## 📊 Phase概要
- Phase名・Phase特性・推定期間・開始日・完了日

## 🎯 Phase成功基準
- 機能要件・品質要件・技術基盤

## 🏢 Phase組織設計方針
- 基本方針・Step別組織構成概要

## 📋 全Step実行プロセス（Step1詳細計画時記録）
- Step構成・各Step概要と期間・組織構成変遷

## 📊 Phase総括レポート（Phase完了時記録）
- 達成度評価・技術的成果・組織効果測定・次Phase申し送り
```

#### StepXX.md内容構造
```markdown
# Step XX 組織設計・実行記録

## 📋 Step概要・🏢 組織設計・🎯 Step成功基準
## 📊 Step実行記録（随時更新）
## ✅ Step終了時レビュー（ADR_013準拠）
- 効率性評価・専門性発揮度・統合調整効率・成果物品質・次Step適応性
```

#### 運用ルール
1. **Phase開始時**: `Phase_Summary.md`・`Step01_Analysis.md` 同時作成
2. **Step1終了時**: 全Step実行プロセスを `Phase_Summary.md` に記録・次Step組織設計ファイル作成
3. **各Step終了時**: `StepXX.md` にレビュー記録・次Step組織設計ファイル作成
4. **Phase完了時**: `Phase_Summary.md` に総括記録・Phase_XXディレクトリ全体移動

#### 新構造の利点
1. **管理性向上**: Step独立管理・参照効率化・並行作業対応
2. **全体概要明確化**: Phase_Summary.mdによる作業範囲・レビュー精度向上
3. **段階的追跡**: Step毎の詳細進捗・組織変遷追跡
4. **再利用性**: Step毎パターン・Phase全体パターンの体系的活用
