# 非機能要件準拠チェック

## 概要

非機能要件（パフォーマンス・セキュリティ・可用性・保守性）が仕様書通りであるかをチェックします。

## チェック項目

### 1. パフォーマンス要件

**目標値**:
- レスポンスタイム: 3秒以内
- 同時接続数: 100ユーザー
- データベースクエリ: N+1問題なし

**チェックリスト**:
- [ ] レスポンスタイムが3秒以内である
- [ ] N+1問題が発生していない
- [ ] 不要なデータベースクエリがない

### 2. セキュリティ要件

**必須対策**:
- パスワードハッシュ化（平文保存禁止）
- SQL Injection対策
- XSS対策
- CSRF対策

**チェックリスト**:
- [ ] パスワードがハッシュ化されている
- [ ] SQL Injection対策済み（パラメータ化クエリ使用）
- [ ] XSS対策済み（入力エスケープ）
- [ ] CSRF対策済み（トークン検証）

### 3. 可用性要件

**必須対応**:
- エラーハンドリング
- ログ記録
- トランザクション管理

**チェックリスト**:
- [ ] try-catchでエラーハンドリングしている
- [ ] エラーログを記録している
- [ ] トランザクションを適切に管理している

### 4. 保守性要件

**必須対応**:
- コメント記載
- ドキュメント整備
- コーディング規約準拠

**チェックリスト**:
- [ ] 仕様書項番がコメントで記録されている
- [ ] ビジネスルールが明確にコメントされている
- [ ] Clean Architecture準拠している
