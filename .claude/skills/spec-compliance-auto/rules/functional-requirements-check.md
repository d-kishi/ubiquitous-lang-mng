# 機能要件準拠チェック

## 概要

機能要件が仕様書通りに実装されているかをチェックします。

## チェック項目

### 1. 実装すべき機能の網羅性

**確認方法**:
- 仕様書の機能リストと実装済み機能を照合
- 未実装機能がないことを確認

**チェックリスト**:
- [ ] 仕様書に記載されている全機能が実装されている
- [ ] 必須機能が全て実装されている
- [ ] オプション機能の実装有無が明確

### 2. 実装してはいけない機能の不在

**確認方法**:
- 否定的仕様（「〜しない」）を洗い出し
- 該当機能が実装されていないことを確認

**チェックリスト**:
- [ ] 否定的仕様をリスト化した
- [ ] 禁止機能が実装されていないことを確認した

### 3. ビジネスルールの正確性

**確認方法**:
- ビジネスルールを定数化
- テストでビジネスルールを検証

**例**:
```csharp
// 仕様書2.3.1準拠: パスワード要件
private const int MinPasswordLength = 8;
private const int MaxPasswordLength = 100;

public bool IsPasswordValid(string password)
{
    return password.Length >= MinPasswordLength &&
           password.Length <= MaxPasswordLength;
}
```

**チェックリスト**:
- [ ] ビジネスルールが仕様書通りである
- [ ] ビジネスルールが定数化されている
- [ ] ビジネスルールがテストで検証されている

### 4. バリデーションロジックの正確性

**確認方法**:
- 入力バリデーションが仕様書通りであることを確認
- エラーメッセージが仕様書通りであることを確認

**チェックリスト**:
- [ ] 必須項目チェックが仕様書通り
- [ ] 形式チェックが仕様書通り（メールアドレス・電話番号等）
- [ ] 範囲チェックが仕様書通り（最小値・最大値）
- [ ] エラーメッセージが仕様書通り
