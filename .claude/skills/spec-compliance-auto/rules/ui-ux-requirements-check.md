# UI/UX要件準拠チェック

## 概要

UI/UX仕様書通りに画面・フォーム・メッセージが実装されているかをチェックします。

## チェック項目

### 1. 画面レイアウト

**チェックリスト**:
- [ ] 画面構成が仕様書通り
- [ ] ボタン配置が仕様書通り
- [ ] ラベル文言が仕様書通り
- [ ] 画面遷移が仕様書通り

### 2. 入力フォームバリデーション

**チェックリスト**:
- [ ] 必須項目マーク（*）が表示されている
- [ ] バリデーションメッセージが仕様書通り
- [ ] リアルタイムバリデーションが仕様通り

**例**:
```razor
@* 仕様書4.2.1準拠: メールアドレス入力フォーム *@
<div class="form-group">
    <label for="email">メールアドレス <span class="required">*</span></label>
    <InputText @bind-Value="Model.Email" class="form-control" />
    <ValidationMessage For="@(() => Model.Email)" />
</div>
```

### 3. エラーメッセージ

**チェックリスト**:
- [ ] エラーメッセージが仕様書通り
- [ ] エラー表示位置が仕様書通り
- [ ] エラー表示スタイルが仕様書通り

**仕様書準拠例**:
```
仕様書: 「メールアドレスを入力してください」
実装: 「メールアドレスを入力してください」 ✅

仕様書: 「メールアドレスを入力してください」
実装: 「メールが必要です」 ❌（仕様逸脱）
```

### 4. ボタン・リンク

**チェックリスト**:
- [ ] ボタンラベルが仕様書通り
- [ ] ボタン配置が仕様書通り
- [ ] 無効化条件が仕様書通り
