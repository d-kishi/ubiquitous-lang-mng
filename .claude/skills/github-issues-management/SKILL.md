# github-issues-management

**バージョン**: 1.0.0
**作成日**: 2025-11-15
**Phase**: Phase B-F2

## 📋 概要

GitHub Issue作成時のラベル判断・運用規則の自律的適用を実現するAgent Skill。

## 🎯 目的

- **ラベル判断ミス防止**: 適切なラベル体系に基づく正確な判断
- **運用規則遵守**: `Doc/08_Organization/Rules/GitHub_Issues運用規則.md` の自律的適用
- **Issue品質向上**: テンプレート活用による標準化
- **作業効率化**: ラベル選択・テンプレート記述の迅速化

## ⏰ 使用タイミング

### 必須使用タイミング

1. **GitHub Issue作成時**（最優先）
   - `gh issue create` コマンド実行前
   - ラベル選択・テンプレート記述時

2. **技術的負債記録時**
   - 技術的負債の発見・分析完了時
   - `/Doc/10_Debt/` からの移行時

3. **課題管理時**
   - アーキテクチャ問題・品質課題の記録時
   - Phase/Step完了時の課題整理時

### 推奨使用タイミング

- Step完了時の技術的負債整理
- Phase振り返り時の課題記録
- 設計レビュー時の問題発見時

## 🚀 基本指針

### 1. 運用規則の必須参照

**原則**: GitHub Issue作成前に必ず `Doc/08_Organization/Rules/GitHub_Issues運用規則.md` を参照

**理由**:
- ラベル体系の正確な理解
- Issue作成基準の確認
- テンプレート構造の把握

### 2. ラベル選択の体系的判断

**3段階判断プロセス**:

```
1. 種別ラベル判断（必須・1つ選択）
   ↓
2. 優先度ラベル判断（必須・1つ選択）
   ↓
3. 影響範囲ラベル判断（任意・複数選択可）
```

**参照**: `label-selection-guide.md`

### 3. Issue作成基準の厳格適用

**作成判断フロー**:

```
1. 作成対象か確認（✅/❌判定）
   ↓
2. 作成判断基準の評価（4基準）
   ↓
3. Issue作成 or Phase計画管理
```

**参照**: `creation-criteria.md`

### 4. テンプレート活用の標準化

**原則**: 基本テンプレートを必ず使用

**カスタマイズ**: 種別ごとの最適化パターン適用

**参照**: `issue-template-patterns.md`

## 📚 補助ファイル

**内容**: ラベル判断基準・選択ロジック

**活用場面**:

- 優先度ラベルの判断（priority/critical, priority/high等）
- 影響範囲ラベルの選択（scope/domain, scope/application等）

### 2. issue-template-patterns.md

**内容**: Issueテンプレート活用パターン

**活用場面**:
- 基本テンプレート構造の理解
- 種別ごとのカスタマイズ
- 必須セクション・任意セクションの判断

### 3. creation-criteria.md

**内容**: Issue作成判断基準

**活用場面**:
- Issue作成対象の判断（✅/❌）
- 作成判断基準の評価（4基準）
- Issue作成 vs Phase計画管理の判断

### 4. label-reference.md

**内容**: ラベル体系一覧（クイックリファレンス）

**活用場面**:
- ラベル体系の迅速な参照
- 利用可能なラベルの確認
- よくある組み合わせパターンの参照
