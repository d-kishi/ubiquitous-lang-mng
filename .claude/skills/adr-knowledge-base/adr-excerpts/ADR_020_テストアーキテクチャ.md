# ADR_020テストアーキテクチャ決定 抜粋

## 重要ポイント

### レイヤー×テストタイプ分離方式

**テストプロジェクト命名規則**（厳守）:
```
UbiquitousLanguageManager.{Layer}.{TestType}.Tests

例:
- UbiquitousLanguageManager.Domain.Unit.Tests
- UbiquitousLanguageManager.Application.Integration.Tests
- UbiquitousLanguageManager.Web.E2E.Tests
```

### 参照関係原則

**Unit Tests**:
- テスト対象レイヤーのみ参照
- 他のレイヤーへの参照禁止

**Integration Tests**:
- 必要な依存層のみ参照
- Infrastructure層はDB接続可

**E2E Tests**:
- 全層参照可
- Playwright使用

### 新規テストプロジェクト作成時の必須確認

1. ADR_020確認（テストアーキテクチャ決定）
2. テストアーキテクチャ設計書確認
3. 新規プロジェクト作成チェックリスト実行
4. 命名規則厳守
5. 参照関係原則遵守
